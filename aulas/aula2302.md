# 🏗️ Projetos Tradicionais vs Projetos de Software

## 🏭 Projetos Tradicionais
- Utilizam **matéria-prima física** (cimento, ferro, madeira, etc.)
- Produto é **tangível**
- Mudanças são difíceis após a construção
- Processo mais rígido
- Exemplo: construção de uma casa 🏠

## 💻 Projetos de Software
- Matéria-prima é **lógica e conhecimento**
- Produto é **intangível**
- Mudanças são mais fáceis (atualizações)
- Processo pode ser iterativo
- Exemplo: sistema de cadastro de produtos

# 🔄 5 Partes da Engenharia de Software

## 1️⃣ Comunicação (Requisitos)
- Levantamento de requisitos
- Entendimento do problema
- Definição das funcionalidades
- Exemplo: sistema deve cadastrar produtos e salvar em CSV

## 2️⃣ Planejamento (Custo e Tempo)
- Definição de cronograma
- Estimativa de esforço
- Definição de recursos necessários

## 3️⃣ Modelagem / Projeto
- Diagrama de Classes (UML)
- Diagrama de Casos de Uso
- Diagrama de Sequência
- Definição da arquitetura (MVC)
- Modelagem das classes:
  - Produto
  - ProdutoDAO
  - TelaCadastro
  - Main

## 4️⃣ Construção
- Implementação em Java
- Uso de Swing para interface gráfica
- Criação da estrutura CSV
- Testes do sistema

## 5️⃣ Entrega
- Sistema funcionando
- Código organizado
- Projeto versionado no GitHub
- Documentação em README.md
