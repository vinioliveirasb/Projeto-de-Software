# 📚 Projeto de Software — Aula 2

## 🧩 Modelagem de Sistemas

Para desenvolver **sistemas de software complexos**, é necessário criar **modelos antes do início do projeto**.

### 🎯 Motivos para usar modelos

- 🧠 **Limitação humana** → sistemas complexos são difíceis de entender sem abstração  
- 💬 **Comunicação** → modelos ajudam os envolvidos a entender o sistema  
- 💰 **Redução de custos** → erros podem ser identificados antes da implementação  
- 🔮 **Previsão de comportamento** → modelos funcionam como um "laboratório" de soluções  

---

# 🧠 Paradigma da Orientação a Objetos

Inspirado por **Alan Kay**, usando uma analogia com organismos vivos.

- Cada elemento do sistema é um **objeto**
- Objetos **interagem por mensagens**
- Cada objeto é uma **unidade autônoma**
- O sistema funciona pela **cooperação entre objetos**

---

# ⚙️ Princípios da Orientação a Objetos

- 📦 Tudo é um **objeto**
- 🔁 Objetos executam tarefas através de **mensagens**
- 🏷️ Objetos pertencem a **classes**
- 🗂️ Classes agrupam **objetos semelhantes**
- 🌳 Classes formam **hierarquias**

---

# 🔄 Conceitos dos Objetos

### ⚡ Operação
Ação que o objeto sabe executar quando solicitada.

### 📩 Mensagem
Pedido enviado a um objeto para executar uma operação.

### 📊 Estado
Conjunto de valores dos atributos do objeto em um momento.

---

# 🧠 Abstração na OO

Focar no **essencial** e ignorar detalhes desnecessários.

Principais conceitos:

- 🔒 **Encapsulamento** → proteção dos dados  
- 🔀 **Polimorfismo** → mesma mensagem, comportamentos diferentes  
- 🌳 **Herança (Generalização)** → reutilização entre classes  
- 🧱 **Composição** → objetos contendo outros objetos  

---

# 📊 UML e Visões do Sistema

### 👤 Visão de Casos de Uso
Mostra a interação entre **usuários e sistema**.

### 🏗️ Visão de Projeto
Define **estrutura e comportamento** do sistema.

### ⚙️ Visão de Implementação
Organiza o sistema em **componentes e subsistemas**.

### 🔄 Visão de Processo
Foca em:

- paralelismo
- sincronização
- desempenho

---

# 📄 Artefatos de Software

Documentos produzidos durante o desenvolvimento utilizando **UML**.

Eles representam as **visões do sistema**.

---

# 🛠️ Etapas do Desenvolvimento

### 💬 Comunicação
Levantamento dos **requisitos**.

### 📅 Planejamento
Definição de **tempo, custo e protótipos**.

### 🧩 Projeto
Modelagem do sistema com **UML**.

### 💻 Construção
Implementação do **código**.

### 🚀 Entrega
Disponibilização do sistema ao usuário.

---

# 🌍 Domínio do Sistema

Área à qual o sistema pertence.

Exemplo:
- sistema bancário → **domínio bancário**

Define o que é **relevante para o sistema**.

---

# 📋 Tipos de Requisitos

### ⚙️ Funcionais
O que o sistema **deve fazer**.

### 📏 Não Funcionais
Como o sistema deve funcionar:

- segurança
- usabilidade
- desempenho

### 📜 Normativos
Regras, leis ou normas que o sistema deve seguir.

---

# 🔍 Fases do Desenvolvimento

### 📊 Análise
Estudo detalhado dos requisitos.

Principais diagramas:
- **Casos de Uso**
- **Classes**

---

### 🏗️ Projeto
Define **como o sistema será implementado**.

Tipos:

- **Projeto de Arquitetura** → organização em subsistemas  
- **Projeto Detalhado** → interação entre objetos  

---

### 💻 Implementação
Tradução dos modelos em **código executável**.

---

### 🧪 Testes

Verificação do sistema conforme as especificações.

#### 🔹 Teste Unitário
Testa partes individuais do código.

#### 🔹 Teste de Integração
Testa a interação entre componentes do sistema.
